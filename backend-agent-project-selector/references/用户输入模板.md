# 用户输入模板

用于让用户更清楚地表达自己的背景、偏好和推荐模式。推荐模式是必填项；如果用户初始化找项目但没有填写推荐模式，必须先完整列出所有可选模式并反问用户选择哪一种模式，不要开始搜索。

## 缺少推荐模式时

必须提示用户可选模式如下，不能只问“你选哪种模式”：

- `agent-only`：只找完整业务型 Agent 项目，适合投 Agent 工程、大模型应用、AI 产品工程方向。
- `backend-only`：只找传统软件后端项目，默认排除 IoT / 硬件接入类，适合投 Java 后端、Go 后端、分布式系统、平台研发方向。
- `mixed`：一个 Agent 项目 + 一个传统软件后端项目，兼顾差异化和后端扎实度。
- `safe-mode`：只推荐容易落地、部署成本低、文档清楚、适合短时间复现的项目。
- `challenge-mode`：推荐更难、更有差异化、更适合冲击高含金量简历的项目。

## 最简模板

用户可以按下面格式填写。推荐模式必须填写，其他缺省项可以写“不限”：

```text
Use $backend-agent-project-selector
推荐模式：mixed
技术栈：Java / Python / Go / 不限
目标岗位：后端实习 / Java 后端 / Agent 工程 / 大模型应用
时间预算：3 天 / 1 周 / 2 周 / 1 个月
背景水平：普通本科 / 985、211 / 有实习 / 无实习 / 不限
项目偏好：业务项目优先 / 技术平台优先 / 新奇 / 扎实 / 两者都要
避开方向：金融 / 电商 / IM / 浏览器插件 / IoT / 硬件接入 / 太难部署 / 不限
输出需求：只要推荐 / 简历写法 / 技术难点挖掘 / 代码验证摘要 / 面试题 / 改造计划 / 全部
```

## 推荐模式

- `agent-only`：只找完整业务型 Agent 项目，适合投 Agent 工程、大模型应用、AI 产品工程方向。
- `backend-only`：只找传统软件后端项目，默认排除 IoT / 硬件接入类，适合投 Java 后端、Go 后端、分布式系统、平台研发方向。
- `mixed`：一个 Agent 项目 + 一个传统软件后端项目，兼顾差异化和后端扎实度。
- `safe-mode`：只推荐容易落地、部署成本低、文档清楚、适合短时间复现的项目。
- `challenge-mode`：推荐更难、更有差异化、更适合冲击高含金量简历的项目。

## 示例 1：只找 Agent 项目

```text
Use $backend-agent-project-selector
推荐模式：agent-only
技术栈：Python
目标岗位：Agent 工程实习
时间预算：2 周
背景水平：普通本科，无实习
项目偏好：完整业务闭环、多 Agent、能讲业务价值
避开方向：浏览器插件、简单聊天机器人、单次 LLM 调用包装
输出需求：推荐 + 技术难点挖掘 + 简历写法 + 改造计划 + 面试题
```

## 示例 2：只找传统软件后端项目

```text
Use $backend-agent-project-selector
推荐模式：backend-only
技术栈：Java / Spring Boot / Redis / MQ
目标岗位：Java 后端实习
时间预算：1 个月
背景水平：有课程项目，无实习
项目偏好：有业务闭环的后端项目、缓存、消息队列、分布式一致性
避开方向：浅层 CRUD、教程复刻、IoT、硬件接入、太冷门的技术栈
输出需求：推荐 + 代码验证摘要 + 技术难点挖掘 + 简历写法 + 改造计划
```

## 输出要求补充

- 如果需要简历写法，默认要求先拉取源码仓库并读取关键代码后再写负责功能。
- 如果用户只允许读 README / 文档，只能输出项目概述和候选分析，不能输出“负责功能 / 技术难点”。

## 示例 3：Agent + 后端混合

```text
Use $backend-agent-project-selector
推荐模式：mixed
技术栈：Java + Python
目标岗位：后端实习，同时想体现 AI 项目经历
时间预算：2-3 周
背景水平：普通本科
项目偏好：一个新奇 Agent 项目，一个扎实后端项目
避开方向：部署特别复杂、没有中文或英文文档的项目
输出需求：全部
```

## 示例 4：稳妥落地模式

```text
Use $backend-agent-project-selector
推荐模式：safe-mode
技术栈：Java
目标岗位：后端实习
时间预算：1 周
背景水平：无实习，需要快速补简历
项目偏好：容易跑通、依赖少、文档清晰、能做小改造
避开方向：多模型 API、GPU、本地大模型、复杂 K8s 部署
输出需求：推荐 + 7 天落地计划 + 简历写法
```

## 示例 5：挑战差异化模式

```text
Use $backend-agent-project-selector
推荐模式：challenge-mode
技术栈：Python / Java / Go 均可
目标岗位：大厂后端实习或 Agent 工程实习
时间预算：1 个月
背景水平：已有一个普通项目，希望做出差异化
项目偏好：多 Agent 协作、复杂业务闭环、调度、评测、可观测性
避开方向：教程项目、普通管理系统、没有业务深度的框架
输出需求：推荐 + 深度改造路线 + 简历写法 + 面试题
```
